# Your First Rust Program

Most programming languages ask you to choose. Languages like Python and
JavaScript keep you safe — they manage memory for you, and you rarely have to
think about how data is stored or cleaned up. But that safety has a cost: your
programs run slower than the hardware allows, and the language sometimes pauses
to reclaim memory you are no longer using.

Languages like C and C++ give you full control and maximum speed — your code
runs with nothing hidden happening behind the scenes. But you are responsible
for managing memory yourself. A single mistake can crash your program, corrupt
data, or produce silently wrong results. These bugs are among the hardest in
all of software to find.

Rust eliminates this trade-off. It gives you the speed of C and the safety of
Python — at the same time, with no runtime cost. The key insight is a set of
rules, checked entirely at compile time, that govern how your program uses
memory. If your Rust program compiles, entire categories of bugs — crashes from
accessing freed memory, data corruption from unsynchronized access, resource
leaks from forgotten cleanup — simply cannot happen. The core idea is
straightforward: every value has exactly one owner, and you can either look at
a value or change it, but not both at the same time. These two principles are
all the compiler needs to keep your programs safe.

This chapter gets you from zero to a running program. By the end, you will have
the Rust toolchain installed, a project created, and your first program
compiled and running. Everything here is the real tooling you will use for
every Rust project going forward — no training wheels.

## Installing Rust

Rust is installed and managed through a tool called _rustup_. It handles
downloading the compiler, the standard library, and the build tool _Cargo_ — and
keeps them all up to date.

Open a terminal and run:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

On Windows, download and run the installer from [https://rustup.rs](https://rustup.rs)
instead. You will also need the Visual Studio C++ build tools, which the
installer will guide you through.

Once installation finishes, verify it worked:

```bash
rustc --version
```

You should see output like `rustc 1.85.0 (4eb161250 2025-02-20)` or newer.
The exact version number does not matter as long as it is 1.85.0 or above —
that is the release that introduced the Rust 2024 edition, which this book
uses throughout.

If you already have Rust installed, make sure you are up to date:

```bash
rustup update
```

## Creating a Project with Cargo

Rust projects are not loose collections of source files. Every Rust project is
managed by Cargo — the build system, package manager, and test runner all in
one. You will use Cargo for everything: creating projects, compiling code,
managing dependencies, running tests, and more.

Create your first project:

```bash
cargo new hello_rust
```

This creates a directory called _hello_rust/_ with the following structure:

```
hello_rust/
├── Cargo.toml
└── src/
    └── main.rs
```

Cargo also initializes a Git repository for you automatically — version
control from the very first line of code.

### The Manifest

Open _Cargo.toml_. This is your project's _manifest_ — the file that tells
Cargo everything it needs to know about your project:

```toml
[package]
name = "hello_rust"
version = "0.1.0"
edition = "2024"

[dependencies]
```

Three fields matter right now:

- `name` is the project name Cargo uses when building your binary.
- `version` follows semantic versioning. Cargo set it to `0.1.0` to signal
  "initial development."
- `edition` tells the compiler which set of language rules to use. Rust
  introduces a new edition every few years to make improvements that would
  otherwise be backward-incompatible. The `"2024"` edition is the latest, and
  it is what `cargo new` selects by default. All code in this book targets
  edition 2024.

An edition is _not_ a new language. All editions compile with the same
compiler, and projects (_crates_, in Rust terminology) using different editions
can depend on each other freely.
An edition is a coherent snapshot of the language's syntax and defaults — a way
to evolve without breaking existing code.

The `[dependencies]` section is empty for now. When you add external libraries
later, they go here.

### The Source File

Open _src/main.rs_:

```rust
fn main() {
    println!("Hello, world!");
}
```

This is a complete Rust program. Let's break it down.

`fn main()` defines a function called `main`. Every executable Rust program
starts here — `main` is the _entry point_, the first function the runtime
calls. The curly braces `{` and `}` delimit the function's body.

`println!("Hello, world!");` prints text to the terminal. The `!` after
`println` means this is a _macro_ call, not a regular function call. For now,
the only thing you need to know about macros is that `println!` is how you
print in Rust and that the `!` is part of its name. The semicolon `;` at the
end marks the end of the _statement_.

## Building and Running

Compile and run your program in one step:

```bash
cargo run
```

You will see output like:

```
   Compiling hello_rust v0.1.0 (/path/to/hello_rust)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.42s
     Running `target/debug/hello_rust`
Hello, world!
```

Cargo compiled your code into a debug binary inside the _target/debug/_
directory, then ran it. The `Hello, world!` line is your program's output.

If you change the code and run **`cargo run`** again, Cargo recompiles only
what changed — it tracks dependencies automatically.

To compile without running, use:

```bash
cargo build
```

The binary appears at _target/debug/hello_rust_ (or _hello_rust.exe_ on
Windows). You can run it directly from the terminal if you want.

## Printing and Formatting

`println!` is more powerful than it first appears. The string you pass to it is
a _format string_ — it can contain placeholders that get replaced with values.

### Basic Placeholders

Use `{}` as a placeholder for values:

```rust
fn main() {
    let language = "Rust";
    let year = 2024;
    println!("Learning {} in {}!", language, year);
}
```

Output: `Learning Rust in 2024!`

Each `{}` is filled in order with the arguments that follow the format string.

### Inline Variables

When a placeholder name matches a variable in scope, you can write the variable
name directly inside the braces:

```rust
fn main() {
    let language = "Rust";
    let year = 2024;
    println!("Learning {language} in {year}!");
}
```

This produces the same output but is easier to read. You will see this style
throughout the book — it is the idiomatic way to format strings in modern Rust.

### Debug Formatting

Some values do not have a user-friendly text representation by default. For
those, use `{:?}` to print their _debug_ representation:

```rust
fn main() {
    let numbers = [1, 2, 3, 4, 5];
    println!("numbers = {:?}", numbers);
}
```

Output: `numbers = [1, 2, 3, 4, 5]`

For complex data, use `{:#?}` to get a _pretty-printed_ version that spreads
the output across multiple lines with indentation. Debug formatting is
invaluable when you are developing and need to inspect what your data looks
like — you will use it constantly.

### Format Specifiers

The colon inside a placeholder introduces a _format specifier_ — instructions
that control how a value is displayed. You have already seen `{:?}` for debug
output. The same colon syntax controls precision, width, alignment, and number
bases:

```rust
fn main() {
    let pi = 3.14159265;
    let label = "pi";
    let count = 42;

    println!("{pi:.2}");         // two decimal places
    println!("{label:>10}");     // right-align in 10 characters
    println!("{count:05}");      // zero-pad to 5 digits
    println!("{count:#x}");      // hexadecimal
    println!("{count:#b}");      // binary
}
```

Output:

```
3.14
        pi
00042
0x2a
0b101010
```

You can combine specifiers: `{pi:>10.2}` means "right-align in a 10-character
field with two decimal places." Besides `>` for right-align, use `<` for
left-align and `^` for center.

The same formatting syntax works in `format!`, which builds a `String` instead
of printing to the screen:

```rust
fn main() {
    let price = 9.99;
    let label = format!("${price:.2}");
    println!("{label}");
}
```

Output: `$9.99`

You will use `format!` throughout Rust whenever you need a formatted string
without immediately printing it.

## Anatomy of a Rust Program

Let's look at a slightly larger example that brings together what you have
learned:

```rust
fn main() {
    let name = "Rustacean";
    let mass_kg = 72.5;
    let is_ready = true;

    println!("Welcome, {name}!");
    println!("Mass: {mass_kg} kg");
    println!("Ready to code: {is_ready}");
    println!("Debug view: {:?}", (name, mass_kg, is_ready));
}
```

Output:

```
Welcome, Rustacean!
Mass: 72.5 kg
Ready to code: true
Debug view: ("Rustacean", 72.5, true)
```

A few things to notice:

- Every statement ends with a semicolon.
- Variables are introduced with `let`. You do not need to declare their types
  — Rust figures out that `name` is a string, `mass_kg` is a floating-point
  number, and `is_ready` is a boolean.
- `println!` is your primary tool for seeing output. Use `{}` with inline
  variable names for clean formatting, `{:?}` when you need to see the
  structure of a value, and format specifiers like `{:.2}` to control
  precision, width, and alignment. Use `format!` when you need a formatted
  `String` without printing it.

## Exercises

These micro-projects let you practice what you have learned — `let` bindings,
`println!`, `format!`, and format specifiers. Each one produces formatted
terminal output. Try to solve them before reading the solution.

### Exercise 1-1: Profile Card

**Goal.** Print a neatly aligned profile card using left-alignment, decimal
precision, and debug formatting.

**Your task:**

1. Bind three variables: a name (string), an age (integer), and a height in
   meters (floating-point).
2. Print a bordered card using box-drawing characters (`╔`, `║`, `╚`, etc.)
   with each field left-aligned inside a fixed-width column.
3. Format the height to exactly two decimal places.
4. Print a debug view of all three values as a tuple using `{:?}`.

> **Key idea.** Left-alignment (`{:<16}`) pads a value with spaces on the
> right so every row lines up. The precision specifier `{:.2}` controls
> decimal places independently of alignment — and you can combine them:
> `{:<16.2}` means "left-align in 16 characters with 2 decimal places."

**Expected output:**

```
╔══════════════════════════╗
║    PROFILE CARD          ║
╠══════════════════════════╣
║ Name:   Ada Lovelace     ║
║ Age:    36               ║
║ Height: 1.65             ║
╚══════════════════════════╝

Debug: ("Ada Lovelace", 36, 1.65)
```

<details>
<summary>Solution</summary>

```rust
fn main() {
    let name = "Ada Lovelace";
    let age = 36;
    let height_m = 1.65;

    println!("╔══════════════════════════╗");
    println!("║    PROFILE CARD          ║");
    println!("╠══════════════════════════╣");
    println!("║ Name:   {name:<16} ║");
    println!("║ Age:    {age:<16} ║");
    println!("║ Height: {height_m:<16.2} ║");
    println!("╚══════════════════════════╝");

    let debug_view = (name, age, height_m);
    println!("\nDebug: {debug_view:?}");
}
```

Each `println!` uses `{:<16}` to left-align the value in a 16-character field,
keeping the right border of the card aligned. The tuple at the end shows `{:?}`
debug formatting — a quick way to inspect multiple values at once.

</details>

### Exercise 1-2: Receipt Formatter

**Goal.** Print a formatted receipt with columns for item name, quantity, unit
price, and line total — all aligned with format specifiers.

**Your task:**

1. Bind three items, each with a name (`&str`), a unit price (`f64`), and a
   quantity (integer).
2. Compute each line total by multiplying price by quantity. Use `as f64` to
   convert the integer quantity for the multiplication.
3. Compute the grand total.
4. Print a receipt with a centered title, a header row, one row per item, and
   the grand total. Use `=` and `-` fill characters for dividers.

> **Key idea.** Fill characters go before the alignment symbol:
> `{:=<42}` fills 42 characters with `=`, left-aligned. `{:^42}` centers
> text in a 42-character field. Combining `{:>7.2}` right-aligns a number
> in a 7-character field with two decimal places — exactly what you need for
> price columns.

**Expected output:**

```
==========================================
                 RECEIPT
==========================================
Item                 Qty   Price    Total
------------------------------------------
Rust Book              2   29.99    59.98
USB Cable              3    7.50    22.50
Stickers              10    2.25    22.50
------------------------------------------
                      Grand Total:  104.98
==========================================
```

<details>
<summary>Solution</summary>

```rust
fn main() {
    let item_1 = "Rust Book";
    let price_1 = 29.99;
    let qty_1 = 2;

    let item_2 = "USB Cable";
    let price_2 = 7.50;
    let qty_2 = 3;

    let item_3 = "Stickers";
    let price_3 = 2.25;
    let qty_3 = 10;

    let total_1 = price_1 * qty_1 as f64;
    let total_2 = price_2 * qty_2 as f64;
    let total_3 = price_3 * qty_3 as f64;
    let grand_total = total_1 + total_2 + total_3;

    println!("{:=<42}", "");
    println!("{:^42}", "RECEIPT");
    println!("{:=<42}", "");
    println!("{:<20} {:>3} {:>7} {:>8}", "Item", "Qty", "Price", "Total");
    println!("{:-<42}", "");
    println!("{:<20} {:>3} {:>7.2} {:>8.2}", item_1, qty_1, price_1, total_1);
    println!("{:<20} {:>3} {:>7.2} {:>8.2}", item_2, qty_2, price_2, total_2);
    println!("{:<20} {:>3} {:>7.2} {:>8.2}", item_3, qty_3, price_3, total_3);
    println!("{:-<42}", "");
    println!("{:>34} {:>7.2}", "Grand Total:", grand_total);
    println!("{:=<42}", "");
}
```

The key formatting tools: `{:<20}` left-aligns item names in a 20-character
column, `{:>7.2}` right-aligns prices with two decimal places, and
`{:=<42}` creates a divider by filling 42 characters with `=`. The `as f64`
on the quantity converts an integer to a float for the multiplication.

</details>

### Exercise 1-3: Color Mixer

**Goal.** Display a color's RGB components in decimal, hexadecimal, and binary
— all from the same values — and build a label string with `format!`.

**Your task:**

1. Bind three `u8` variables for red, green, and blue components (e.g.,
   `255`, `128`, `0`).
2. Bind a color name as a `&str`.
3. Print the color in decimal `rgb(R, G, B)` format.
4. Print the hex representation as `#rrggbb` using zero-padded, two-digit
   lowercase hex (`{:02x}`).
5. Print each component right-aligned in a 3-character field.
6. Print each component's 8-bit binary representation using `{:08b}`.
7. Use `format!` to build a label string combining the name and hex value,
   then print it.
8. Print a debug view of the three components as a tuple.

> **Key idea.** The same integer value can be displayed in completely
> different ways depending on the format specifier: `{:02x}` for hex,
> `{:08b}` for binary, `{:>3}` for padded decimal. The `format!` macro
> uses the exact same specifier syntax as `println!` but returns a
> `String` instead of printing — useful whenever you need to build a
> formatted value for later use.

**Expected output:**

```
Color: Rust Orange

  Decimal:  rgb(255, 128, 0)
  Hex:      #ff8000
  Padded:   R=255  G=128  B=  0

Bit patterns:
  R: 11111111
  G: 10000000
  B: 00000000

Label: Rust Orange (#ff8000)
Debug: (255, 128, 0)
```

<details>
<summary>Solution</summary>

```rust
fn main() {
    let red: u8 = 255;
    let green: u8 = 128;
    let blue: u8 = 0;

    let color_name = "Rust Orange";

    println!("Color: {color_name}");
    println!();
    println!("  Decimal:  rgb({red}, {green}, {blue})");
    println!("  Hex:      #{red:02x}{green:02x}{blue:02x}");
    println!("  Padded:   R={red:>3}  G={green:>3}  B={blue:>3}");
    println!();
    println!("Bit patterns:");
    println!("  R: {red:08b}");
    println!("  G: {green:08b}");
    println!("  B: {blue:08b}");

    let label = format!("{color_name} (#{red:02x}{green:02x}{blue:02x})");
    println!();
    println!("Label: {label}");

    let components = (red, green, blue);
    println!("Debug: {components:?}");
}
```

The same three `u8` values appear in four representations: decimal (default),
hexadecimal (`{:02x}` — zero-pad to 2 digits, lowercase hex), binary
(`{:08b}` — zero-pad to 8 bits), and right-aligned decimal (`{:>3}`). The
`format!` call on the label demonstrates building a `String` without printing
it — the same specifiers work in both `println!` and `format!`.

</details>
