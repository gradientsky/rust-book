# Your First Rust Program

Most programming languages ask you to choose. You can have a language that
protects you from crashes, memory leaks, and security holes — but it will be
slow, or it will pause unpredictably while a garbage collector runs. Or you can
have a language that runs as fast as the hardware allows — but one wrong pointer
and your program corrupts memory, leaks secrets, or silently produces wrong
answers.

Rust refuses this trade-off. It gives you the performance of C and C++ and the
safety guarantees of languages like Java or Python — at the same time, with no
runtime overhead. It does this through a set of rules checked entirely at
compile time. If your program compiles, an entire category of bugs — dangling
pointers, data races, use-after-free, buffer overflows — simply cannot happen.

This chapter gets you from zero to a running program. By the end, you will have
the Rust toolchain installed, a project created, and your first program
compiled and running. Everything here is the real tooling you will use for
every Rust project going forward — no training wheels.

## Installing Rust

Rust is installed and managed through a tool called rustup. It handles
downloading the compiler, the standard library, and the build tool Cargo — and
keeps them all up to date.

Open a terminal and run:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

On Windows, download and run the installer from [https://rustup.rs](https://rustup.rs)
instead. You will also need the Visual Studio C++ build tools, which the
installer will guide you through.

Once installation finishes, verify it worked:

```bash
rustc --version
```

You should see output like `rustc 1.85.0 (4eb161250 2025-02-20)` or newer.
The exact version number does not matter as long as it is 1.85.0 or above —
that is the release that introduced the Rust 2024 edition, which this book
uses throughout.

If you already have Rust installed, make sure you are up to date:

```bash
rustup update
```

## Creating a Project with Cargo

Rust projects are not loose collections of source files. Every Rust project is
managed by _Cargo_ — the build system, package manager, and test runner all in
one. You will use Cargo for everything: creating projects, compiling code,
managing dependencies, running tests, and more.

Create your first project:

```bash
cargo new hello_rust
```

This creates a directory called _hello_rust/_ with the following structure:

```
hello_rust/
├── Cargo.toml
└── src/
    └── main.rs
```

Cargo also initializes a Git repository for you automatically — version
control from the very first line of code.

### The Manifest

Open _Cargo.toml_. This is your project's _manifest_ — the file that tells
Cargo everything it needs to know about your project:

```toml
[package]
name = "hello_rust"
version = "0.1.0"
edition = "2024"

[dependencies]
```

Three fields matter right now:

- `name` is the project name Cargo uses when building your binary.
- `version` follows semantic versioning. Cargo set it to `0.1.0` to signal
  "initial development."
- `edition` tells the compiler which set of language rules to use. Rust
  introduces a new edition every few years to make improvements that would
  otherwise be backward-incompatible. The `"2024"` edition is the latest, and
  it is what `cargo new` selects by default. All code in this book targets
  edition 2024.

An edition is _not_ a new language. All editions compile with the same
compiler, and crates using different editions can depend on each other freely.
An edition is a coherent snapshot of the language's syntax and defaults — a way
to evolve without breaking existing code.

The `[dependencies]` section is empty for now. When you add external libraries
later, they go here.

### The Source File

Open _src/main.rs_:

```rust
fn main() {
    println!("Hello, world!");
}
```

This is a complete Rust program. Let's break it down.

`fn main()` defines a function called `main`. Every executable Rust program
starts here — `main` is the _entry point_, the first function the runtime
calls. The curly braces `{` and `}` delimit the function's body.

`println!("Hello, world!");` prints text to the terminal. The `!` after
`println` means this is a _macro_ call, not a regular function call. For now,
the only thing you need to know about macros is that `println!` is how you
print in Rust and that the `!` is part of its name. The semicolon `;` at the
end marks the end of the _statement_.

## Building and Running

Compile and run your program in one step:

```bash
cargo run
```

You will see output like:

```
   Compiling hello_rust v0.1.0 (/path/to/hello_rust)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.42s
     Running `target/debug/hello_rust`
Hello, world!
```

Cargo compiled your code into a debug binary inside the _target/debug/_
directory, then ran it. The `Hello, world!` line is your program's output.

If you change the code and run **`cargo run`** again, Cargo recompiles only
what changed — it tracks dependencies automatically.

To compile without running, use:

```bash
cargo build
```

The binary appears at _target/debug/hello_rust_ (or _hello_rust.exe_ on
Windows). You can run it directly from the terminal if you want.

## Printing and Formatting

`println!` is more powerful than it first appears. The string you pass to it is
a _format string_ — it can contain placeholders that get replaced with values.

### Basic Placeholders

Use `{}` as a placeholder for values:

```rust
fn main() {
    let language = "Rust";
    let year = 2024;
    println!("Learning {} in {}!", language, year);
}
```

Output: `Learning Rust in 2024!`

Each `{}` is filled in order with the arguments that follow the format string.

### Inline Variables

When a placeholder name matches a variable in scope, you can write the variable
name directly inside the braces:

```rust
fn main() {
    let language = "Rust";
    let year = 2024;
    println!("Learning {language} in {year}!");
}
```

This produces the same output but is easier to read. You will see this style
throughout the book — it is the idiomatic way to format strings in modern Rust.

### Debug Formatting

Some values do not have a user-friendly text representation by default. For
those, use `{:?}` to print their _debug_ representation:

```rust
fn main() {
    let numbers = [1, 2, 3, 4, 5];
    println!("numbers = {:?}", numbers);
}
```

Output: `numbers = [1, 2, 3, 4, 5]`

For complex data, use `{:#?}` to get a _pretty-printed_ version that spreads
the output across multiple lines with indentation. Debug formatting is
invaluable when you are developing and need to inspect what your data looks
like — you will use it constantly.

## Anatomy of a Rust Program

Let's look at a slightly larger example that brings together what you have
learned:

```rust
fn main() {
    let name = "Rustacean";
    let version = 1.85;
    let is_ready = true;

    println!("Welcome, {name}!");
    println!("Minimum compiler version: {version}");
    println!("Ready to code: {is_ready}");
    println!("Debug view: {:?}", (name, version, is_ready));
}
```

Output:

```
Welcome, Rustacean!
Minimum compiler version: 1.85
Ready to code: true
Debug view: ("Rustacean", 1.85, true)
```

A few things to notice:

- Every statement ends with a semicolon.
- Variables are introduced with `let`. You do not need to declare their types
  — Rust figures out that `name` is a string, `version` is a floating-point
  number, and `is_ready` is a boolean.
- `println!` is your primary tool for seeing output. Use `{}` with inline
  variable names for clean formatting, and `{:?}` when you need to see the
  structure of a value.

You now have a working Rust environment, a project managed by Cargo, and the
ability to write, build, run, and print output from Rust programs. In the next
chapter, you will meet Rust's most distinctive feature for newcomers — a
compiler that does not just reject bad code, but _tells you exactly how to fix
it_.
